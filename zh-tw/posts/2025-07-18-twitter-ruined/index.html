Twitter 亂了套 - 馬斯克接手後的服務故障

2025-07-18 個人
閱讀時間大約 1 分鐘

Available in English  

（我是真的氣到，專門寫一個 blog post 噴）

某天我在更改 Display Name，發現得到 500 內部錯誤。在網上搜尋後，我才發現事情沒有想得這麼簡單……

Twitter 的名稱分為兩種： Handle 是唯一的，如重複就無法設置，這是服務的基礎機制。但是可以重複的 Display Name，沒有理由出現錯誤……

網路上排查，發現這個問題來源已久，但近期似乎舊方法都解決不了。

Link to Community Discussio

一開始 X Staff 還搞不清楚狀況，以為是 Handle name 的問題。但就如之前提到，Display name 不同於 Handle name，沒有唯一性。

文章內有人建議嘗試過 autoclicker，試圖暴力通過，我只得到 429 （笑哭）。

我寫了一個簡單的小程式，你也可以自己試試看

from time import sleep
from pynput.mouse import Button, Controller

mouse = Controller()
sleep(2)
last_pos = None
count = 0

while True:
  if mouse.position != last_pos:
    print(f"mosue: {mouse.position}, last: {last_pos}")
    break
  if last_pos == None: last_pos = mouse.position

  mouse.press(Button.left)
  mouse.release(Button.left)
  sleep(0.005)
  count+=1

print(count)

這是個應該告知使用者的限制

就如同字數限制，假設哪天X 把字數限制的界面從軟體裡刪除，然後超過字數就發 500，會引發多大的反響。

這是應該明確寫在軟體上、告知使用者的功能，卻一點都沒有跟。

類似的事情還有發生過，例如前陣子的貼文載入限制。一天最多只能載入多少貼文。（現在似乎沒限制這麼嚴格）

大家都有嘗試過聯繫X，但卻只有得到制式文章，沒有任何聯絡方法。

就在前幾天，X Staff 以「非技術問題」直接關閉了長達3年討論的論壇。關閉後提供的方法也沒有一個是可以聯絡到官方、解決問題的。 這似乎明顯有掩蓋事件的嫌疑。

而這些事情都是發生在 Elon Msuk 接手 Twitter 實行他的政策之後，不能排除內部政策對這次事件的影響性。而且從始至終官方都沒有對這次的事件做出任何正面回應。

（事件開始於 2023/2，Musk 於 2022/4 接手）

說真的，不修問題也不正面回應就算了……在Elon Musk 的政策下，軟體裡放一堆 Premium 廣告完全不能解決問題

我只能說，我們真的需要這種獨裁者當作這個平台的掌管者嗎？
